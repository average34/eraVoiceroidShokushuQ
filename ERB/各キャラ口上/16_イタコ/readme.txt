【導入方法】
このreadme.txtが入っているフォルダ「16_イタコ」を
触手宮のフォルダ/ERB/各キャラ口上
の直下に配置してください

その状態でニューゲームするとイタコがある程度喋るようになります


【簡易説明】
東方触手宮、ずん子（きりたん(因幡てゐ)）の口上改変版です
ライセンスは全ての改変元である因幡てゐのライセンスに準じます

概ねのセリフが改変元のずん子口上がですわですわちゅわちゅわ言うだけです。力尽きました。
タコ姉さまの口調の参考にEXボイス一覧、ずんちゃんといっしょ等を拝見しましたが、
あんまりｺｯｼｮﾘなことの参考にはなりませんでしたので違和感があると思います。
途中から製作者の口上モードが淫乱状態にバグったため、
口上ごとに温度差があるかと思います。

刻印って何かに使うのかな？と思ってED分岐に使用しました
複数の分岐条件を満たしている時に条件を満たしている中からどれか一つをランダムに選出
とかしたかったけど諦め


Q：なんで作りましたの？
A：東北三姉妹を触手宮にぶち込んでパーティを組ませたかった。

Q：イタコはこんな喋り方しませんわよ？
A：どうしたらそれっぽいか教えてくれると捗ります。

Q：ずん子口上のEDとセリフ色が違いますわ！
A：システムとの区別がつきにくかったので勝手に変えました。
 　気になる場合はご自分で調整をお願いいたします 　

Q：エラーがありましたわ
A：教えてくださると助かります。治せるかはわかりません。

========================================
東方触手宮、因幡てゐの口上改変版です
ライセンス読む限りでは改変、公開しても問題無いはず……
ライセンスは全て因幡てゐのライセンスに準じます

改変箇所は更新履歴を参照して下さい
今後も増えるような増えないようなです

運用するにあたって
eraVoiceroid触手宮には
ERBフォルダ内の関数フォルダに東方触手宮にあるCRC32_UNI.ERBが存在しないためエラーが発生します
しますので、CRC32_UNI.ERBを同梱したので関数フォルダに突っ込むか、ERBフォルダがある所にERBフォルダを突っ込めば勝手に関数フォルダに入ります

※既知のバグ＆バグ（？）
１．着替え時に本体側のアイテム説明を参照している為、チャイナドレスに着替えると「この服は美鈴が普段着ている服とは似ているが少し違うようだ」と言うメッセージが出る
直す権限が無い場所なのでスルーしてくだしあ、回避方法は10分ほど模索して諦めました

いくつかありそうな質問に対する回答
Q：きりたんの各キャラクターに対する呼び方を変更したい

　 好感度が高くなったら○○お姉ちゃんって呼ばせたり出来るぞ
 　当然の話ですが、行為中にここの呼称が出て来たりするのでそこも加味して変更してください
　 

Q：きりたんの口調はこんなのじゃない！
A：わかるー（途中で脳がバグった）、これを改変するなり、1から素晴らしい口上を作るなりして欲しい

Q：淫乱付いたら口調がはっちゃけすぎでは？
A：きりたんは変わってしまった……（改変時にきりたんらしさよりエロさを取った結果です）

Q：エラーが出たぞ！　バグだああああああ！
A：自力で直して下さい、重大過ぎるバグの場合は教えてくれたら直せたら直して再配布します
　 プログラミングの勉強も全くしてない素人 of 素人なので直せなかったらごめんね

== 独自エスケープシーケンス & 特殊文字 =========================================
\b  表示フォント色をBAD表示の赤に
    \b～\bで挟まれた範囲の文章をBAD色(赤)で表示します
\c  一行文のCLEARLINEを実行します
\d  表示フォント色をシステム標準色に
    \d～\dで挟まれた範囲の文章をシステム標準色で表示します
\g  表示フォント色をGOOD表示の青に
    \g～\gで挟まれた範囲の文章をGOOD色(青)で表示します
\n  改行
\m  地の文キャンセルフラグ
    システムメッセージで表示される地の文を無効化します
\M  地の文を表示
    このエスケープを利用した場合
    システム標準で描画される地の文は表示されず、任意の場所で表示されるようになります
\w  WAIT付改行 PRINTFORM -W の動作
♥  ハート表示(文字色:赤)
    環境による差を吸収する為ぱにめーしょんを利用、別途関数呼び出しに置き換えます


== 文字入力 特殊処理 ===========================================================
KOJO_27_0_ADD_MESSAGE経由での口上は
引数に半角スペースとタブを渡しても無視されます
その為、下記の形で複数行記述が可能です

{
	KOJO_27_0_ADD_MESSAGE, @"動作テスト1\n
	                         動作テスト2\n"
}



== 関数名規約 ==================================================================

-- 前回値参照関数 --------------------------------------------------------------
	@~SYSTEM_MESSAGE~の呼び出しに成功した場合
	自動的に@~SYSTEM_MESSAGE~_PREV_~、又は@~SYSTEM_MESSAGE~_PREV_PURE_~が呼び出されます
	通常の口上が未定義でPREV関数のみを定義した場合、呼び出されないので注意
	PREVのみを定義したい場合は、空関数でいいので通常の口上を定義してください
	
	自動で呼び出されるのはあくまで一つ前の値を参照した関数となる為
	それ以前の値を参照したい場合は明示的に呼び出しを行ってください
